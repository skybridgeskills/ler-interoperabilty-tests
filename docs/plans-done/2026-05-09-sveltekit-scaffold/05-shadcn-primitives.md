# Phase 05 — shadcn-svelte primitives & stories

## Scope of phase

Land the day-one UI primitive set and one Storybook story per primitive.
After this phase, all six primitives (`Button`, `Card`, `Badge`, `Input`,
`Tabs`, `Dialog`) render correctly under Tokyo Night light + dark, are
exported from their respective `index.ts` files, and have a `*.stories.svelte`
that exercises their variants.

Files created in this phase:

- `components.json` (shadcn-svelte config) at repo root.
- `src/lib/components/ui/button/{Button.svelte, button-variants.ts, index.ts, Button.stories.svelte}`.
- `src/lib/components/ui/card/{Card.svelte, index.ts, Card.stories.svelte}` (+ subcomponents
  `CardHeader`, `CardTitle`, `CardDescription`, `CardContent`, `CardFooter` per shadcn).
- `src/lib/components/ui/badge/{Badge.svelte, badge-variants.ts, index.ts, Badge.stories.svelte}`.
- `src/lib/components/ui/input/{Input.svelte, index.ts, Input.stories.svelte}`.
- `src/lib/components/ui/tabs/{Tabs.svelte, TabsList.svelte, TabsTrigger.svelte, TabsContent.svelte, index.ts, Tabs.stories.svelte}`.
- `src/lib/components/ui/dialog/{Dialog.svelte, DialogTrigger.svelte, DialogContent.svelte, DialogHeader.svelte, DialogTitle.svelte, DialogDescription.svelte, DialogFooter.svelte, DialogClose.svelte, index.ts, Dialog.stories.svelte}`.
- Update `ThemeToggle.svelte` to use the new `Button` primitive (the
  Phase 03 inline `<button>` was a placeholder).

## Code Organization Reminders

- One primitive per directory; the directory name is `kebab-case` even
  though the component file is PascalCase.
- Variants live in `<name>-variants.ts` (Button, Badge). Other primitives
  inline `tv()` calls are fine when there's only one variant cluster.
- Each primitive's `index.ts` re-exports the component and any
  subcomponents.
- Stories: `<Component>.stories.svelte` next to the component; one
  `defineMeta()` block per file with multiple `<Story>` blocks for
  variant coverage.

## Style conventions

- **Generated by `pnpm dlx shadcn-svelte add <name>`** when feasible —
  the CLI scaffolds primitives matching shadcn-svelte conventions; only
  hand-edit when the generated code conflicts with our token names. This
  happens primarily because shadcn-svelte's defaults assume the standard
  `--primary`/`--accent`/etc. token set, which we already have under our
  Tokyo Night names. Generated code should largely work as-is; only the
  variant strings (e.g. `bg-primary text-primary-foreground`) need a
  glance.
- File / variant naming follows shadcn-svelte conventions verbatim.
- Stories use `@storybook/addon-svelte-csf` v5 (`<script module>` +
  `defineMeta` + `<Story>`). One story file per primitive with at least
  variants for each defined variant in `tv()`.
- Imports follow external → `$lib/` → relative. shadcn-svelte's generated
  files alphabetize within each group.
- The `cn()` helper from `$lib/utils.ts` is the only acceptable way to
  merge classes inside primitives; do not concatenate strings or use
  template literals for class lists.
- Phase 05 is UI-only; no providers/factories/schemas in scope.

## Implementation Details

### `components.json`

```json
{
	"$schema": "https://shadcn-svelte.com/schema.json",
	"style": "default",
	"tailwind": {
		"css": "src/routes/layout.css",
		"baseColor": "slate"
	},
	"aliases": {
		"components": "$lib/components",
		"utils": "$lib/utils",
		"ui": "$lib/components/ui",
		"hooks": "$lib/hooks",
		"lib": "$lib"
	},
	"typescript": true,
	"registry": "https://shadcn-svelte.com/registry"
}
```

### Primitive generation

```sh
pnpm dlx shadcn-svelte@latest add button card badge input tabs dialog
```

Review each generated file. Token references should already match our
palette (`bg-primary`, `text-foreground`, `border-input`, `ring-ring`,
etc.) because we kept the standard token names in `layout.css`. If
shadcn-svelte's generator deviates, hand-edit the variant strings.

### `Button.stories.svelte` (template for all primitives)

```svelte
<script module lang="ts">
	import { defineMeta } from '@storybook/addon-svelte-csf';

	import { Button } from './index.js';

	const { Story } = defineMeta({
		title: 'UI/Button',
		component: Button,
		parameters: {
			docs: {
				description: {
					component:
						'Default shadcn-svelte Button rendered against Tokyo Night tokens. ' +
						'Variant + size combinations should remain legible in both light and dark modes.'
				}
			}
		}
	});
</script>

<Story name="Variants">
	<div class="flex flex-wrap gap-3 bg-background p-6">
		<Button variant="default">Default</Button>
		<Button variant="secondary">Secondary</Button>
		<Button variant="destructive">Destructive</Button>
		<Button variant="outline">Outline</Button>
		<Button variant="ghost">Ghost</Button>
		<Button variant="link">Link</Button>
	</div>
</Story>

<Story name="Sizes">
	<div class="flex flex-wrap items-center gap-3 bg-background p-6">
		<Button size="sm">Small</Button>
		<Button>Default</Button>
		<Button size="lg">Large</Button>
		<Button size="icon" aria-label="icon button">⌘</Button>
	</div>
</Story>

<Story name="Disabled">
	<div class="flex gap-3 bg-background p-6">
		<Button disabled>Default</Button>
		<Button variant="outline" disabled>Outline</Button>
		<Button variant="destructive" disabled>Destructive</Button>
	</div>
</Story>
```

Apply the same shape to `Card`, `Badge`, `Input`, `Tabs`, `Dialog`. Each
gets:

- One "all variants" story.
- A "states" story for interactive primitives (focus, disabled, with
  helper text for `Input`; open/closed for `Dialog`).
- Tabs gets a representative three-tab example (Wallet / Verifier /
  Issuer makes sense — foreshadows the IA without committing).
